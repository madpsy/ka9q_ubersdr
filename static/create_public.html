Welcome to the Public Instance Registry

📡 What is Instance Reporting?

Instance reporting allows your UberSDR to be listed in the public registry at instances.ubersdr.org, making it discoverable by other radio enthusiasts worldwide.

✅ Benefits

  • Share your SDR with the global amateur radio community
  • Contribute to propagation studies and band condition monitoring
  • Help others learn about SDR technology
  • Join a network of public SDR receivers

🌐 Ways to Make Your Instance Public

🔌 Port Forwarding with Caddy

Forward ports 80 and 443 to your UberSDR host in your router. Caddy (included in Docker setup) acts as a reverse proxy.

💡 Automatic HTTPS: When TLS is enabled in config.yaml with a valid domain and email, Caddy automatically obtains and renews Let's Encrypt certificates.

☁️ Cloudflare Tunnel

If you have your own domain configured in Cloudflare you can create a tunnel. No port forwarding required!

💡 Port Forwarding Setup

If not using Cloudflare Tunnel, you need to configure port forwarding on your router.

Option 1: Using Your Public IP Address (HTTP only)
  • Keep "Automatically use my public IP address" checked
  • Forward port 80 to your UberSDR host IP address in your router
  • Users will connect via http://your-public-ip
  • Note: HTTPS/certificates are not available with IP addresses
Option 2: Using a Domain Name (HTTPS with automatic certificates)
  • Uncheck "Automatically use my public IP address"
  • Enter your domain name (e.g., sdr.example.com) in the hostname field
  • Enable "Use TLS/HTTPS for public connections"
  • Forward ports 80 and 443 to your UberSDR host IP address
  • Caddy will automatically obtain and renew Let's Encrypt certificates

Important: For Option 2, your domain must point to your public IP address for certificate generation to work.
